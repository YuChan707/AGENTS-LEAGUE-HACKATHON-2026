==============================================================================
 OnLooker - DICCIONARIO DE ENTIDADES (DTOs / marshmallow Schemas)
==============================================================================
 Convenciones de tipo:
   UUID     -> identificador unico (relaciones por ID)
   String   -> texto libre
   Enum     -> String restringido a un vocabulario controlado (validate.OneOf)
   Integer  -> numero entero
   Float    -> numero decimal
   Date     -> fecha
   Nested   -> objeto anidado (otra entidad)
   List[x]  -> lista de elementos de tipo x
   Dict     -> mapa clave/valor flexible
==============================================================================


##############################################################################
# 1) DATA INGESTORS  (dtos/data_ingestors.py) - ENTIDADES DE ENTRADA
##############################################################################

------------------------------------------------------------------------------
 VOCABULARIOS CONTROLADOS (enums)
------------------------------------------------------------------------------
 GENDERS                  : male | female | non_binary | other
 EDUCATION_LEVELS         : none | primary | secondary | high_school |
                            associate | bachelor | master | doctorate
 INCOME_BRACKETS          : low | lower_middle | middle | upper_middle | high
 PARTICULARITY_DIMENSIONS : gender | ethnicity | income | field


==============================================================================
 DATA CRUDA  (extraida de Data Commons)
==============================================================================

------------------------------------------------------------------------------
 GeoPoint
------------------------------------------------------------------------------
 Punto geografico simple. Centroide de una ubicacion.
   latitude   : Float   (-90 a 90)
   longitude  : Float   (-180 a 180)

------------------------------------------------------------------------------
 LocationStatistics
------------------------------------------------------------------------------
 Metadata estadistica de una ubicacion. Separada de la identidad para tener
 responsabilidad unica y poder versionar/anidar.
   unemployment_rate     : Float    % de desempleo (0-100)
   poverty_rate          : Float    % bajo linea de pobreza (0-100)
   cost_of_living_index  : Float    indice de costo de vida (100 = media nac.)
   median_income         : Integer  ingreso mediano anual (USD)
   avg_household_size     : Float    tamano medio del hogar
   safety_index          : Float    indice de seguridad (0-100)

------------------------------------------------------------------------------
 Location
------------------------------------------------------------------------------
 Identidad de una ubicacion (foco NY, USA). Une identidad + estadistica.
   location_id    : UUID      ID interno para relacionar (busqueda barata)
   dcid           : String    Data Commons ID (ej: "geoId/36")
   location_name  : String    nombre claro del sitio
   coordinates    : Nested    GeoPoint (centroide)
   zip_code       : String    codigo postal
   country        : String    pais
   state          : String    estado
   city           : String    ciudad
   statistics     : Nested    LocationStatistics
   data_source    : String    procedencia de la data (default datacommons.org)
   last_updated   : Date      fecha de ultima actualizacion

------------------------------------------------------------------------------
 AgeRange
------------------------------------------------------------------------------
 Rango etario reutilizable (un rango NO es un solo numero).
   min_age : Integer  (0-120)
   max_age : Integer  (0-120)

------------------------------------------------------------------------------
 DemographicGroup
------------------------------------------------------------------------------
 Grupo demografico extraido de la data cruda. Referencia a Location por ID.
   group_id          : UUID      identidad del grupo
   group_name        : String    nombre del grupo
   location_id       : UUID      referencia a Location
   age_range         : Nested    AgeRange
   gender            : Enum      GENDERS
   ethnicity         : String    etnia
   education_level   : Enum      EDUCATION_LEVELS
   income_bracket    : Enum      INCOME_BRACKETS
   median_income     : Integer   ingreso mediano del grupo (USD)
   population_size   : Integer   tamano del grupo (>= 0)
   population_share  : Float      proporcion dentro de la location (0-1)


==============================================================================
 DATA SINTETICA  ($$ generada con LLM)
==============================================================================

------------------------------------------------------------------------------
 SegmentParticularity
------------------------------------------------------------------------------
 Particularidad de un segmento, parametrica por "dimension". Reemplaza a las
 antiguas Gender/Ethnicity/Income/FieldParticularities en una sola entidad.
   particularity_id : UUID      identidad
   dimension        : Enum      PARTICULARITY_DIMENSIONS
   label            : String    ej: "female", "latino", "middle", "tech"
   description      : String    explicacion generada por la LLM
   attributes       : Dict       atributos libres segun la dimension
   traits           : List[Str]  rasgos clave en lenguaje natural
   generated_by     : String    trazabilidad (default "llm")

------------------------------------------------------------------------------
 PsychographicGroup
------------------------------------------------------------------------------
 Grupo psicografico: valores, intereses, estilo de vida, actitudes.
   group_id            : UUID       identidad
   group_name          : String     nombre
   values              : List[Str]   valores
   interests           : List[Str]   intereses
   lifestyle           : List[Str]   estilo de vida
   personality_traits  : List[Str]   rasgos de personalidad
   motivations         : List[Str]   motivaciones
   attitudes           : List[Str]   actitudes
   generated_by        : String     trazabilidad (default "llm")

------------------------------------------------------------------------------
 BehavioralGroup
------------------------------------------------------------------------------
 Grupo conductual: como se comportan online (conocimiento general de la LLM).
   group_id                    : UUID        identidad
   group_name                  : String      nombre
   preferred_platforms         : List[Str]    ej: ["instagram", "linkedin"]
   content_format_preferences  : List[Str]    ej: ["video", "carrusel"]
   peak_activity_hours         : List[Int]    horas pico (0-23)
   avg_session_minutes         : Float        duracion media de sesion
   engagement_style            : String       ej: "lurker", "creator", "sharer"
   price_sensitivity           : Float        sensibilidad al precio (0-1)
   sharing_propensity          : Float        propension a compartir (0-1)
   generated_by                : String       trazabilidad (default "llm")


##############################################################################
# 2) DATA PROCESSORS  (dtos/data_processors.py) - ENTIDADES DE SALIDA
##############################################################################

------------------------------------------------------------------------------
 VOCABULARIOS CONTROLADOS (enums)
------------------------------------------------------------------------------
 ASSET_TYPES : document | presentation | spreadsheet | pdf | other

------------------------------------------------------------------------------
 AudienceSegment
------------------------------------------------------------------------------
 Segmento de audiencia accionable = demografia + particularidades + psico +
 conducta. Une todo por referencias de ID (no nesteo pesado).
   segment_id             : UUID        identidad
   label                  : String      nombre legible del segmento
   location_id            : UUID        referencia a Location
   demographic_group_id   : UUID        referencia a DemographicGroup
   psychographic_group_id : UUID        referencia a PsychographicGroup
   behavioral_group_id    : UUID        referencia a BehavioralGroup
   particularity_ids      : List[UUID]  referencias a SegmentParticularity
   estimated_reach        : Integer     personas alcanzables (>= 0)
   confidence             : Float        confianza del armado (0-1)

------------------------------------------------------------------------------
 ProjectAsset
------------------------------------------------------------------------------
 El archivo M365 que OnLooker analiza.
   asset_id           : UUID        identidad
   name               : String      nombre del archivo
   asset_type         : Enum        ASSET_TYPES
   summary            : String      resumen del contenido
   language           : String      idioma del documento
   target_segment_ids : List[UUID]  audiencias objetivo (AudienceSegment)
   source_uri         : String      ubicacion del archivo (M365 / Drive)

------------------------------------------------------------------------------
 ReactionProfile
------------------------------------------------------------------------------
 Salida de la LLM: como reacciona un segmento a un asset/seccion.
   reaction_id            : UUID        identidad
   asset_id               : UUID        a que asset reacciona
   segment_id             : UUID        que audiencia reacciona
   section                : String      seccion/slide/parrafo (opcional)
   sentiment_score        : Float        sentimiento (0-1)
   comprehension_score    : Float        comprension (0-1)
   cultural_fit_score     : Float        encaje cultural (0-1)
   engagement_likelihood  : Float        probabilidad de engagement (0-1)
   strengths              : List[Str]    que funciona bien
   risks                  : List[Str]    que puede fallar / ofender / confundir
   recommendations        : List[Str]    como mejorarlo para este segmento
   confidence             : Float        confianza (0-1)
   generated_by           : String      trazabilidad (default "llm")

==============================================================================
 FLUJO DE RELACIONES (resumen)
==============================================================================
 Location 1---* DemographicGroup
 AudienceSegment --> Location, DemographicGroup, PsychographicGroup,
                     BehavioralGroup, [SegmentParticularity]
 ProjectAsset --> [AudienceSegment]
 ReactionProfile --> ProjectAsset + AudienceSegment
==============================================================================
